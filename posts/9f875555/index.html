Vue3中模板指令的非兼容变更

按键修饰符

同时废弃了全局config.keyCodes选项

key属性

  1. v-if/v-else/v-else-if的key不再是必须的，vue3.x会自动生成唯一key
    不可以通过手动提供key的方式，来强制重用分支

  2. <template v-for>的key应该设置在<template>标签上，而不是设置在他的子结点上

v-if和v-for的优先级调整

这个可太棒了

v-bind现在对排序敏感（v-bind的合并行为）

如果在一个元素上同时定义了v-bind=”object”和一个相同的单独的property
那么v-bind的绑定会被覆盖

在vue3.x中 v-bind和单独的property有排序关系，看代码

v-on 的 .native 修饰符已被移除

v-for中的ref不再注册ref数组

在 Vue 2 中，在 v-for 里使用的 ref attribute 会用 ref 数组填充相应的 $refs property。当存在嵌套的 v-for 时，这种行为会变得不明确且效率低下。

在 Vue 3 中，这样的用法将不再在 $ref 中自动创建数组。要从单个绑定获取多个 ref，请将 ref 绑定到一个更灵活的函数上 (这是一个新特性)

itemRefs 不必是数组：它也可以是一个对象，其 ref 会通过迭代的 key 被设置。

如果需要，itemRef 也可以是响应式的且可以被监听。
